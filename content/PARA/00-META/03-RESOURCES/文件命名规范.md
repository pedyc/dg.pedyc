## 文件夹与文件命名规范

### 1. 总则

- 命名应简洁明了，突出内容主题，便于检索。
- 统一采用 " 中文 + 领域描述 " 方式，避免拼音、缩写和无意义编号。
- 禁止出现特殊字符（如? * < > | \ /: "），避免系统兼容性问题。

### 2. 文件夹命名规范

- 一级目录：直接反映 PARA 分类（如 "10-PROJECTS"、"20-AREAS"、"30-RESOURCES"、"40-ARCHIVES"）。
- 二级目录：用中文描述具体领域或主题，如 " 前端工程化 "、" 算法与数据结构 "、" 职业规划 "。
- 三级目录及以下：仅在必要时使用，尽量减少嵌套层级。
- 示例：
	- 20-AREAS/专业技术/算法真题
	- 20-AREAS/专业技术/算法总结
	- 20-AREAS/专业技术/前端工程化

### 3. 文件命名规范

- 采用中文 + 内容描述，避免无意义数字或英文缩写。
- 题库、题单、总结等建议加前缀区分，如 "00 算法真题库.md"、"10 经典问题题单.md"、" 广度优先搜索 (BFS).md"。
- 归档类文件建议加 " 归档 " 字样。
- 示例：
	- 00 算法真题库.md
	- 10 经典问题题单.md
	- 广度优先搜索 (BFS).md
	- 前端框架与生态.md
- 参见 [[领域命名规范]]

### 4. 其他建议

- 定期复查目录和文件命名，发现不规范及时调整。
- 新增内容前先查阅本规范，确保命名一致性。
- 对于历史遗留命名不规范的内容，逐步迁移和重命名。

如有特殊命名需求，可在本规范下补充说明。

### PARA 命名规范

- 10-PROJECTS：当前进行中的项目，建议以 " 年份 + 季度 + 主题 " 命名，便于检索。
- 20-AREAS：持续关注的专业领域，建议减少嵌套层级，统一命名规范（如 " 前端工程化 "、" 算法与数据结构 "）。
- 30-RESOURCES：常用工具、学习资料、知识清单，优先补充前端开发相关内容。
- 40-ARCHIVES：已归档项目、历史资料。
