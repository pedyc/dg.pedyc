设计元数据系统的核心原则是：==为了检索和自动化，而不是为了分类而分类==
元数据模板参加：[[template_meta]]

## 核心属性 type（笔记是什么类型？）

可以通过 AI 或者 Templater 插件自动生成

| 取值 (Value) | 含义 (Definition) | 适用场景示例                                               |
| ---------- | --------------- | ---------------------------------------------------- |
| concept    | 概念/观点           | 你的原子笔记主力。例如：" 双重编码理论 "、" 必要难度 "、" 熵增 "。              |
| term       | 术语/定义           | 纯粹的名词解释，不包含观点。例如："HTTP 协议 "、" 多巴胺 "、"ROI"。           |
| source     | 来源/输入           | 读书笔记、文章摘要、视频记录。例如："《思考快与慢》读书笔记 "。                    |
| moc        | 地图/索引           | Map of Content，你的 MOC 页。例如："0300 视觉思维 "、" 前端开发路线图 "。 |
| person     | 人物              | 记录某个人的观点或联系方式。例如："Tiago Forte"、" 某客户 "。              |
| meeting    | 会议/事件           | 带有时间戳的记录。例如："2025-10-12 产品复盘会 "。                     |
| procedure  | 流程/SOP          | 指导如何做某事的步骤。例如：" 服务器部署流程 "、" 周回顾 SOP"。                |

## 状态属性 status（笔记处于什么阶段？）

通过 Dataview 删选 " 待处理笔记 "

| 取值 (Value)  | 含义     | 对应的行动                                     |
| ----------- | ------ | ----------------------------------------- |
| fleeting    | 闪念/草稿  | 刚捕捉的想法，尚未整理。行动： 需要加工或删除。                  |
| processing  | 加工中    | 正在阅读的书、正在写的文章。行动： 需要持续投入时间。               |
| cultivating | 培育中    | 笔记已建立，但还需要补充案例或反向链接。行动： 以后遇到相关内容记得链接过来。   |
| done        | 完成/原子化 | 观点已闭环，格式已规范，放入 Zettelkasten 存档。行动： 仅作为引用。 |

## 关联属性 tags & up（笔记处于什么位置？）

- `tags` (标签)： 用于**横向**的、非结构化的属性。
	- _用法：_ `#心理学`, `#工具/Obsidian`, `#写作素材`
	- _原则：_ 标签主要用于 " 话题（Topic）" 或 " 媒介（Medium）"。
- `up` (父级链接)： 这是一个非常有用的高级技巧（类似 Breadcrumbs 插件逻辑）。
	- _用法：_ 指向该笔记所属的 **MOC** 或 **Project** 笔记。
	- _示例：_ 在笔记《双重编码理论》中，`up: [[0300视觉思维]]`。
	- _价值：_ 这样你即便不看文件夹，也能通过 Dataview 自动生成层级目录。
