本库是一个遵循 [[PARA笔记组织法]] 的个人知识管理库。

## 本库核心概念

### 项目 (Projects)

* **核心特征**：有明确起止时间和目标。
* **生命周期**：短期。
* **判断标准**：能用动词描述（完成/实现 XX）。
* **元数据设计**：

| 类型     | 字段              | 说明             | 示例/备注                                                                                                      |
| ------ | --------------- | -------------- | ---------------------------------------------------------------------------------------------------------- |
| **项目** | `title`         | 项目名称（动词 + 名词）  | " 开发用户登录系统 "                                                                                               |
|        | `status`        | 待办/进行中/已完成/已取消 |                                                                                                            |
|        | `date-created`  | 项目创建日期         | 2025-04-23                                                                                                 |
|        | `date-modified` | 最后修改日期         |                                                                                                            |
|        | `date-due`      | 项目截止日期         |                                                                                                            |
|        | `area`          | 相关领域（双向链接）     | [00前端性能优化领域](app://obsidian.md/00%E5%89%8D%E7%AB%AF%E6%80%A7%E8%83%BD%E4%BC%98%E5%8C%96%E9%A2%86%E5%9F%9F) |
|        | `tags`          | 标签             | [#重要](app://obsidian.md/index.html#%E9%87%8D%E8%A6%81)                                                     |
|        | `description`   | 项目描述           |                                                                                                            |
|        | `相关资源`          | 资源标签           | [#资源](app://obsidian.md/index.html#%E8%B5%84%E6%BA%90)                                                     |

* **模板**：[[template_project]]

> 注：每个项目都是单独的一个文件，例如 `[[2025Q1-清理待办列表]]`，表示的是 `时间维度-完成事项`，存储在 `10-PROJECTS` 文件夹

### 领域 (Areas)

* **核心特征**：需长期维护的责任领域。
* **生命周期**：持续。
* **判断标准**：是否涉及持续标准（保持/提升 XX）。
* **元数据设计**：

| 类型     | 字段              | 说明         | 示例/备注       |
| ------ | --------------- | ---------- | ----------- |
| **领域** | `title`         | 领域名称       | " 前端性能优化 "  |
|        | `status`        | 活跃/维护中/已废弃 |             |
|        | `date-created`  | 领域创建日期     |             |
|        | `date-modified` | 最后修改日期     |             |
|        | `tags`          | 标签         | [area/core] |
|        | `description`   | 领域描述       |             |

> 注：领域是 PARA 中的中转站，用于指导项目，也用于链接资源。本库中的领域以单个文件表示，例如 `[[00专业技术领域]]`，`00` 用于表示此文件的层级，更显眼也更易于检索，子级领域表示为 `[[01前端工程化领域]]`，表示 `专业技术领域/前端工程化领域`

### 资源 (Resources)

* **核心特征**：潜在有用的参考资料。
* **生命周期**：不确定。
* **判断标准**：是否可能在未来被多个场景调用。
* **元数据设计**：

| 类型     | 字段              | 说明     | 示例/备注                                                  |
| ------ | --------------- | ------ | ------------------------------------------------------ |
| **资源** | `title`         | 资源名称   | " 性能优化检查清单 "                                           |
|        | `date-created`  | 资源创建日期 |                                                        |
|        | `date-modified` | 最后修改日期 |                                                        |
|        | `tags`          | 标签     | [#模板](app://obsidian.md/index.html#%E6%A8%A1%E6%9D%BF) |
|        | `description`   | 资源描述   |                                                        |

### 归档 (Archives)

* **核心特征**：已完成/失效的内容。
* **生命周期**：冻结。
* **判断标准**：是否 6 个月内未被使用。
* **元数据设计**：

| 类型     | 字段              | 说明                 | 示例/备注                                                  |
| ------ | --------------- | ------------------ | ------------------------------------------------------ |
| **归档** | `title`         | 原名称（YYYYMM- 原名称格式） | 202504- 旧项目文档                                          |
|        | `date-archived` | 归档日期               |                                                        |
|        | `tags`          | 标签                 | [#历史](app://obsidian.md/index.html#%E5%8E%86%E5%8F%B2) |
|        | `description`   | 内容描述               |                                                        |

## 本库目录结构设计

```bash
PKM/
├── 00-META/                 # 系统元数据
├── 10-PROJECTS/             # 项目（保持原结构）
│   ├── 搁置/
│   └── 完成/
│       └── 2025Q1-学习Docker/  # 建议：项目细化文件夹
├── 20-AREAS/
│   ├── 个人成长/            # 领域
│   ├── 人工智能/            # 领域
│   └── 知识管理/            # 领域 → 仅保留子领域
│       ├── 00-CORE/         # 核心方法论（需持续迭代）
│       │   ├── 知识组织方案/
│       │   └── 视觉思维/    # 子领域（含项目/资源）
│       ├── 01-WORKFLOWS/    # 工作流标准
│       └── 02-STRATEGY/     # 战略工具
├── 30-RESOURCES/            # 全局资源库（原分散资源集中）
│   ├── 效率工具/
│   │   └── PARA工具包/      # 从领域迁移至此
│   ├── 分类法/
│   ├── FAQ库/               # 从领域迁移至此
│   └── 视觉思维/            # 通用模板/教程
│       └── 思维分类.excalidraw
└── 40-ARCHIVES/             # 归档
```
