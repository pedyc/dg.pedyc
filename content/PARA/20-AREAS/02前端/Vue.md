## 背景

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其他大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

## 核心思想

Vue 的核心思想是**数据驱动**和**组件化**。

- **数据驱动**：通过声明式地将数据绑定到 DOM，简化了手动操作 DOM 的过程，提高了开发效率。
- **组件化**：将页面拆分成独立、可复用的组件，提高了代码的可维护性和复用性。

## 模型

Vue 的核心模型包括：

- [[虚拟 DOM（Vue）]]：Vue 使用虚拟 DOM 来追踪状态变化并优化 DOM 操作。
- [[响应式系统（Vue）]]：Vue 的响应式系统使得数据变化可以自动更新到视图。
- [[组件系统（Vue）]]：Vue 的组件系统允许开发者构建可复用的 UI 元素。
- [[生态系统（Vue）]]：Vue 的生态系统允许开发者调用额外能力。

## 原理

1. **响应式原理**：
		- Vue 2.x 使用 `Object.defineProperty` 来劫持数据的 `getter` 和 `setter`，当数据发生变化时，自动通知相关的依赖进行更新。
		- Vue 3.x 使用 `Proxy` 对象来实现响应式，可以更高效地监听数据的变化。
2. **虚拟 DOM**：
		- Vue 首先将模板编译成渲染函数 (render function)，该函数返回一个虚拟 DOM 树。
		- 当数据发生变化时，Vue 会重新执行渲染函数，生成新的虚拟 DOM 树。
		- 然后，Vue 会比较新旧虚拟 DOM 树的差异，并将这些差异应用到真实的 DOM 上，从而实现高效的更新。
3. **组件化**：
		- Vue 组件由模板、脚本和样式组成。
		- 组件可以嵌套使用，形成组件树。
		- Vue 通过组件的生命周期钩子函数来管理组件的状态和行为。

## 相关理论与概念

- [[React]]: 另一个流行的 JavaScript 框架，与 Vue 有很多相似之处。
- [[Angular]]: 一个由 Google 开发的完整的前端框架。
- [[Virtual DOM]]: 一种用于优化 DOM 操作的技术。
- [[MVVM]]: 一种软件架构模式，Vue 采用了 MVVM 模式。
