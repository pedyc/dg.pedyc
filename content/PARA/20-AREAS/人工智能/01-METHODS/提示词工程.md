## [[提示词工程]]

### 概念

提示词工程（Prompt Engineering）是一种设计和优化提示词的技术，用于引导大型语言模型（LLM）生成期望的输出。提示词是输入给 LLM 的文本，用于指示模型执行特定任务或生成特定类型的内容。

### 核心原则

1. **清晰明确：** 提示词应该清晰明确地表达你的意图，避免歧义和模糊。
2. **具体详细：** 提示词应该提供足够的上下文信息，帮助 LLM 理解你的需求。
3. **结构化：** 提示词可以使用结构化的格式，例如列表、表格等，提高可读性和可理解性。
4. **迭代优化：** 提示词需要不断迭代优化，根据 LLM 的输出结果进行调整。

### 常用技巧

1. **指令 (Instructions)：**
		* 使用动词明确指示 LLM 执行的任务，例如 " 生成 "、" 总结 "、" 翻译 " 等。
		* 使用限定词限制 LLM 的输出范围，例如 " 用简洁的语言 "、" 不超过 100 字 " 等。
2. **上下文 (Context)：**
		* 提供背景信息，帮助 LLM 理解你的需求。
		* 提供相关示例，帮助 LLM 学习你的风格和偏好。
3. **输入数据 (Input Data)：**
		* 提供 LLM 需要处理的数据，例如文本、代码、图像等。
		* 使用分隔符清晰地分隔输入数据和指令。
4. **输出格式 (Output Format)：**
		* 明确指定 LLM 的输出格式，例如 JSON、Markdown、HTML 等。
		* 使用示例展示期望的输出格式。
5. **角色扮演 (Role Play)：**
		* 让 LLM 扮演特定角色，例如 " 你是一位专业的翻译 "、" 你是一位资深的前端工程师 " 等。
		* 可以提高 LLM 的专业性和创造力。
6. **思维链 (Chain of Thought)：**
		* 引导 LLM 逐步推理，展示思考过程。
		* 可以提高 LLM 的逻辑性和准确性。
7. **Few-shot Learning：**
		* 提供少量示例，让 LLM 学习如何完成任务。
		* 适用于任务比较复杂或数据比较稀缺的情况。

### 提示词模板

1. **问题回答：**

```bash
问题：{问题}
答案：
```

2. **文本摘要：**

```bash
请总结以下文本：
{文本}
摘要：

```

3. **代码生成：**

```bash
请生成以下代码：
{代码描述}
代码：

```

4. **翻译：**

```bash
请将以下文本翻译成 {目标语言}：
{文本}
翻译：
```

### 提示词策略

1. **分解复杂任务：** 将复杂任务分解为多个简单的子任务，逐步完成。
2. **利用 LLM 的优势：** 充分利用 LLM 的知识和推理能力，提高输出质量。
3. **避免 LLM 的弱点：** 避免让 LLM 处理不擅长的任务，例如需要精确计算的任务。
4. **持续迭代优化：** 不断尝试不同的提示词，找到最佳的提示词组合。

### 评估指标

1. **准确性：** LLM 的输出是否准确地回答了问题或完成了任务。
2. **相关性：** LLM 的输出是否与提示词相关。
3. **流畅性：** LLM 的输出是否自然流畅，易于理解。
4. **创造性：** LLM 的输出是否具有创造性，能够提供新的视角或想法。

### 应用场景

1. **内容创作：** 自动生成文章、博客、故事等。
2. **代码生成：** 自动生成代码、文档、测试用例等。
3. **智能客服：** 自动回答用户问题，提供技术支持。
4. **教育：** 自动生成练习题、试卷、教学材料等。
5. **搜索：** 提高搜索结果的准确性和相关性。

### 局限性

1. **偏见：** LLM 可能会受到训练数据中的偏见影响，生成带有偏见的输出。
2. **幻觉：** LLM 可能会生成不真实或不合理的内容。
3. **安全：** LLM 可能会被用于生成恶意内容，例如虚假信息、网络钓鱼等。

### 未来发展趋势

1. **自动化提示词工程：** 利用 AI 技术自动设计和优化提示词。
2. **多模态提示词工程：** 支持图像、音频、视频等多种输入模态。
3. **个性化提示词工程：** 根据用户画像和偏好，生成个性化的提示词。
