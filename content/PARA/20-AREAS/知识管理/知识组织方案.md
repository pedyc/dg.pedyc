## 一、目录结构设计

```plaintext
_scripts  #脚本目录
_template  #模板目录
Clippings  #Obsidian浏览器插件剪藏目录
Excalidraw  #Excalidraw插件绘图目录
Follow  #Follow剪藏目录
SimpRead  #简悦浏览器插件剪藏目录
PKM/
├── 00-META/                 # 系统元数据
├── 10-PROJECTS/             # 项目（保持原结构）
│   ├── 搁置/
│   └── 完成/
│       └── 2025Q1-学习Docker/  # 建议：项目细化文件夹
├── 20-AREAS/
│   ├── 个人成长/            # 领域
│   ├── 人工智能/            # 领域
│   └── 知识管理/            # 领域 → 仅保留子领域
│       ├── 00-CORE/         # 核心方法论（需持续迭代）
│       │   ├── 知识组织方案/
│       │   └── 视觉思维/    # 子领域（含项目/资源）
│       ├── 01-WORKFLOWS/    # 工作流标准
│       └── 02-STRATEGY/     # 战略工具
├── 30-RESOURCES/            # 全局资源库（原分散资源集中）
│   ├── 效率工具/
│   │   └── PARA工具包/      # 从领域迁移至此
│   ├── 分类法/
│   ├── FAQ库/               # 从领域迁移至此
│   └── 视觉思维/            # 通用模板/教程
│       └── 思维分类.excalidraw
└── 40-ARCHIVES/             # 归档
```
