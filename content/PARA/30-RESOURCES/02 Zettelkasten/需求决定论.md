在经济学理论中，需求决定论是说只有先让人有意愿买（有需求），企业才会去生产。

其核心逻辑是：**需求创造供给**。

在计算机科学领域中，这一理论同样成立，企业会**因为需求去推进技术**，例如 Google 公司的核心业务是搜索引擎，他们为了搜索业务能够长远发展，事实上成为了互联网标准的推动者之一；亚马逊 AWS 云业务一开始也是因为亚马逊内部因为重复造轮子浪费了大量生产力，才会催生出通过公有云 API 调用资源的云计算业务。

而对于个人来说，需求推动能力成长同样是个人成长的驱动力之一，这种成长模式通常遵循以下路径：

> **遭遇真实痛点 $\rightarrow$ 现有能力失效 $\rightarrow$ 被迫学习新技能 $\rightarrow$ 能力跃迁**。

例如我在构建 [[知识管理概述|个人知识管理]] 系统时遭遇的痛点如笔记分类问题、信息过载问题等，推动我去研究 [[PARA笔记法]] 和 [[卡片盒笔记法|卡片盒笔记法]]，最终构建了具有个人特色的混合型知识库。
