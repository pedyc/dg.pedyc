## PARA 实施指南

### 1. PARA 方法简介

PARA（Projects, Areas, Resources, Archives）是一种高效的知识管理与任务组织方法，适用于数字笔记、项目管理和个人成长。

- **Projects（项目）**：有明确目标和截止时间的任务集合。
- **Areas（领域）**：持续关注和维护的责任领域。
- **Resources（资源）**：可复用的参考资料、工具和知识。
- **Archives（归档）**：已完成或暂时搁置的信息。

### 2. PARA 目录结构建议

- 10-PROJECTS：当前进行中的项目，建议以 " 年份 + 季度 + 主题 " 命名，便于检索。
- 20-AREAS：持续关注的专业领域，建议减少嵌套层级，统一命名规范（如 " 前端工程化 "、" 算法与数据结构 "）。
- 30-RESOURCES：常用工具、学习资料、知识清单，优先补充前端开发相关内容。
- 40-ARCHIVES：已归档项目、历史资料。

### 3. 前端开发相关 PARA 实践

- 在 20-AREAS 下设立 " 前端工程化 "、" 前端性能优化 "、" 算法与数据结构 " 等子目录，避免多层嵌套。
- 在 30-RESOURCES 下补充前端开发常用工具、框架、学习资源（如 React、Vue、Webpack、Vite、TypeScript、前端效率工具等）。
- 统一文件夹和文档命名风格，建议使用中文 + 领域描述。

### 4. 目录优化建议

- 检查并减少 20-AREAS 下的多层嵌套，将知识点归类到一级或二级目录。
- 规范命名，如 " 算法与数据结构 " 下的 " 算法真题 "、" 算法总结 " 均采用统一风格。
- 30-RESOURCES 可增补 " 前端效率工具 "、" 常用组件库 "、" 学习路线 " 等子目录。

### 5. PARA 实施流程

1. 明确每条笔记或文档属于 PARA 哪一类。
2. 定期复盘，调整目录结构，确保检索高效。
3. 新增知识优先归入 Resources，项目推进归入 Projects，持续维护内容归入 Areas。
