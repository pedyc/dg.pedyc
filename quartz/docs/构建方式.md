- postscript 的构建方式 ： postscript.js 主要通过 componentResources.ts 中的 joinScripts(componentResources.afterDOMLoaded) 构建。它收集了所有组件的 afterDOMLoaded 脚本以及全局页面资源（包括 SPA 路由脚本、Popover 脚本和各种分析工具的动态加载逻辑）。这些脚本被合并、包裹在 IIFE 中，并通过 esbuild 进行最小化处理，最终在 DOM 加载完成后执行。其内容侧重于页面交互、动态功能和按需加载的分析脚本。

- inline 文件的构建方式 ： inline 文件通过 esbuild.config.mjs 中的 inline-script-loader 插件处理 .inline.(ts|js) 文件，以及通过 sassPlugin 处理 Sass/CSS 文件。这些文件经过转译、打包、最小化和 tree shaking 后，其内容直接内联到最终的 HTML 页面中。 inline 文件通常包含关键路径 CSS 和首屏渲染所需的少量 JavaScript，旨在最大限度地减少首次内容绘制
